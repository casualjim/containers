# Chisel Ubuntu 26.04 Migration Implementation Plan

## Overview

Single-version bump of all chisel-based container images from Ubuntu 25.10 (Questing) to Ubuntu 26.04 LTS (Resolute Raccoon). Updates `UBUNTU_RELEASE` and `CHISEL_VERSION` variables, syncs upstream chisel slices, rewrites custom LLVM libc++ slices from LLVM 20 to LLVM 22, and updates all documentation references. No target changes, no CI changes — 25.10 support is dropped entirely.

## Desired End State

Build command unchanged:

```bash
docker buildx bake static libc libc-ssl libcxx libcxx-ssl \
  libcxx-ssl-tesseract libcxx-ssl-ladybug lbug-cli bun fission-bun
```

Tags reflect 26.04:

```
ghcr.io/casualjim/bare:static-26.04-{BUILD_NUMBER}
ghcr.io/casualjim/bare:libcxx-26.04-{BUILD_NUMBER}
...
```

Image contents: LLVM 22.1.2 libraries at `/usr/lib/*-linux-gnu/` with SONAME symlinks created by `install-libcxx.sh`.

## What We're NOT Doing

- rustbuilder (stays on 24.04)
- netdebug (out of scope)
- sqlx-cli (Alpine-based, unaffected)
- Dual-version builds
- CI workflow restructuring
- Automated tests

## Phase 1: Chisel Infrastructure

### Overview
Update the four infrastructure files that drive the chisel-based image build: `UBUNTU_RELEASE` and `CHISEL_VERSION` in docker-bake.hcl and Dockerfile, chisel archive configuration in chisel.yaml, and sync script defaults. This phase does NOT touch custom slices — after this phase, the `static` target should build as a canary to validate the chisel toolchain before custom slices are rewritten.

### Changes Required

#### 1. docker-bake.hcl — UBUNTU_RELEASE and CHISEL_VERSION
**File**: `docker-bake.hcl`
**Changes**: Update variable defaults on lines 2 and 5-6. Cascades through 28 tag expressions and the `chisel-common` args block automatically.

```hcl
variable "UBUNTU_RELEASE" {
  default = "26.04"
}

variable "CHISEL_VERSION" {
  default = "v1.4.1"
}
```

#### 2. Dockerfile — ARG defaults
**File**: `Dockerfile`
**Changes**: Update build argument defaults on lines 1-2. Drives `FROM ubuntu:${UBUNTU_RELEASE}` and the chisel binary download URL.

```dockerfile
ARG UBUNTU_RELEASE=26.04
ARG CHISEL_VERSION=v1.4.1
```

#### 3. chisel-releases/chisel.yaml — Archive configuration
**File**: `chisel-releases/chisel.yaml`
**Changes**: Update format to v3 (required by upstream 26.04 slices), version to 26.04, suites from questing to resolute, maintenance timeline. The GPG key block (`ubuntu-archive-key-2018`) remains unchanged.

```yaml
format: v3

maintenance:
  standard: 2026-04-23
  expanded: 2031-05-29
  legacy: 2036-04-23
  end-of-life: 2038-04-27

archives:
  ubuntu:
    version: 26.04
    components: [main, universe]
    suites: [resolute, resolute-security, resolute-updates]
    public-keys: [ubuntu-archive-key-2018]
```

#### 4. sync-chisel-slices.sh — Default RELEASE and CUSTOM_PACKAGES
**File**: `sync-chisel-slices.sh`
**Changes**: Update hardcoded default on line 7 from 25.10 to 26.04. Update CUSTOM_PACKAGES array (lines 53-57) from 4 LLVM 20 entries to 3 LLVM 22 entries. The REQUIRED_PACKAGES array is unchanged.

```bash
UBUNTU_RELEASE="${UBUNTU_RELEASE:-26.04}"
```

```bash

  "libc++1"
  "libc++abi1"
  "llvm-libunwind1"
```

## Phase 2: Custom LLVM 22 Slices

### Overview
Delete the three LLVM 20 custom slices (packages don't exist in 26.04), create two new slices for the renamed packages (`libc++abi1`, `llvm-libunwind1`), and rewrite the `libc++1` meta-slice to delegate to the new LLVM 22 runtime slices. After this phase, the `libcxx` target should build and libraries should resolve.

### Changes Required

#### 5. chisel-releases/slices/libc++abi1.yaml — NEW
**File**: `chisel-releases/slices/libc++abi1.yaml`
**Changes**: New slice for the unversioned `libc++abi1` package (from llvm-toolchain-22, 1:22.1.2). Essentials: `[libc6_libs, libgcc-s1_libs]` — follows deb metadata (no `llvm-libunwind1` dependency). Single path: `/usr/lib/*-linux-gnu/libc++abi.so.1*`.

```yaml
package: libc++abi1

essential:
  - libc++abi1_copyright

slices:
  libs:
    essential:
      - libc6_libs
      - libgcc-s1_libs
    contents:
      /usr/lib/*-linux-gnu/libc++abi.so.1*:

  copyright:
    contents:
      /usr/share/doc/libc++abi1/copyright:
```

#### 6. chisel-releases/slices/llvm-libunwind1.yaml — NEW
**File**: `chisel-releases/slices/llvm-libunwind1.yaml`
**Changes**: New slice for the renamed `llvm-libunwind1` package (from llvm-toolchain-22, 1:22.1.2). Essentials: `[libc6_libs, libgcc-s1_libs]`. Single path: `/usr/lib/*-linux-gnu/libunwind.so.1*`.

```yaml
package: llvm-libunwind1

essential:
  - llvm-libunwind1_copyright

slices:
  libs:
    essential:
      - libc6_libs
      - libgcc-s1_libs
    contents:
      /usr/lib/*-linux-gnu/libunwind.so.1*:

  copyright:
    contents:
      /usr/share/doc/llvm-libunwind1/copyright:
```

#### 7. chisel-releases/slices/libc++1.yaml — REWRITE
**File**: `chisel-releases/slices/libc++1.yaml`
**Changes**: Rewrite the meta-slice. Previously delegated to `libc++1-20_libs`. Now lists runtime dependencies directly: `libc++abi1_libs`, `libc6_libs`, `llvm-libunwind1_libs` — all three as peers (per Option A decision).

```yaml
package: libc++1

slices:
  libs:
    essential:
      - libc++abi1_libs
      - libc6_libs
      - llvm-libunwind1_libs
```

#### 8-10. Delete old LLVM 20 slices
**Files**: `chisel-releases/slices/libc++1-20.yaml`, `chisel-releases/slices/libc++abi1-20.yaml`, `chisel-releases/slices/libunwind-20.yaml`
**Changes**: Delete these files. The packages `libc++1-20`, `libc++abi1-20`, and `libunwind-20` do not exist in Ubuntu 26.04.

## Testing Strategy

### Automated:
None — this repository has no automated image tests. The build success/failure itself is the primary automated gate (CI runs `docker buildx bake` on PRs).

### Manual Testing Steps:
1. `docker buildx bake static` — validates chisel toolchain + base slices (Phase 1 canary)
2. `docker buildx bake libcxx` — validates custom LLVM 22 slices (Phase 2 canary)
3. `docker run --rm --entrypoint /bin/bash ghcr.io/casualjim/bare:libcxx -c "ldd /usr/lib/*-linux-gnu/libc++.so.1"` — library resolution
4. `docker buildx bake libcxx-ssl libcxx-ssl-tesseract libcxx-ssl-ladybug lbug-cli bun fission-bun` — full matrix
5. `docker run --rm ghcr.io/casualjim/bun:latest --version` — bun runtime
6. `docker run --rm ghcr.io/casualjim/lbug-cli:latest lbug --version` — ladybug CLI
7. Repeat steps 1-2 for `--set *.platform=linux/arm64` — cross-arch validation
8. Compare image sizes (`docker images`) against 25.10 baselines

## Performance Considerations

- Image sizes may differ slightly from 25.10 baselines — compare after building
- No runtime performance impact — this is a build-time/library-version change
